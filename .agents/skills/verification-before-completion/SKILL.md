# 完成前验证

## 铁律

```
不运行验证，就不能声称完成
```

## 验证门禁

声称任何状态前：

1. **确定**：什么命令能证明这个声明？
2. **运行**：执行完整的命令（新鲜的、完整的）
3. **读取**：完整输出，检查退出码，统计失败数
4. **核实**：输出是否确认声明？
5. **只有在此之后**：才能做出声明

## Rust 项目的验证命令

| 声明 | 需要 |
|------|------|
| 编译通过 | `cargo build --release` 输出: exit 0 |
| 测试通过 | `cargo test` 输出: 0 失败 |
| 无警告 | `cargo build 2>&1 \| grep warning` → 空 |
| Bug 修复 | 原来失败的测试现在通过 |
| 性能无退化 | `make test` 全部通过 |
| 数据清理 | `du -sh data/` → 空或不存在 |
