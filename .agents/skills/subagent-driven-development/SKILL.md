# 子代理驱动开发

## 核心原则

每个任务派生子代理 → 任务审查（规格合规 + 代码质量）→ 最终全分支审查 = 高质量快速迭代

## 流程

1. **读取计划**，记录上下文和全局约束
2. **逐个派生子代理** 执行任务，每个任务完成后审查
3. **发现 Critical/Important 问题** → 派修复子代理 → 重新审查
4. **Minor 问题** → 记录到进度账本
5. **全部任务完成** → 最终全分支代码审查
6. **使用 finishing-a-development-branch** 完成开发

## 红牌

- 永远不要跳过任务审查
- 永远不要在未修复问题的情况下继续
- 永远不要并行派发多个实现子代理
- 永远不要信任子代理报告而不独立验证
